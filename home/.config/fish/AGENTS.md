# FISH SHELL CONFIG

Layered: `config.fish` (minimal) → `conf.d/*.fish` (auto-sourced, alphabetical) →
`functions/` + `completions/` (lazy). Plugins via **fisher**. Ported from `.zshrc`.

> **Agents:** put new setup in the right `conf.d/` fragment, never in `config.fish`.
> Load order is **alphabetical** — that's why alias overrides live in `zz-aliases.fish`.

## STRUCTURE

```
fish/
├── config.fish             # Minimal: disables greeting only. NO tool inits here.
├── fish_plugins            # fisher manifest (jorgebucaran/fisher, jhillyerd/plugin-git)
├── conf.d/                 # Auto-sourced, alphabetical order
│   ├── env.fish            # LANG, EDITOR, MANPATH, JAVA_HOME, ghostty TERM
│   ├── paths.fish          # PATH via `fish_add_path --move` (brew first, .bun last)
│   ├── fzf.fish            # FZF_DEFAULT_COMMAND + `fzf --fish | source`
│   ├── zoxide.fish         # `zoxide init fish | source`
│   ├── starship.fish       # prompt (same starship.toml as zsh)
│   ├── onepassword.fish    # `op completion fish` (plugins.sh is zsh-only)
│   ├── vite-plus.fish      # Vite+ PATH + `vp` env wrapper
│   ├── git.fish            # (fisher/plugin-git — GITIGNORED, not ours)
│   └── zz-aliases.fish     # aliases; `zz-` prefix so it loads AFTER plugin-git
├── functions/              # fisher/plugin-git functions (GITIGNORED, `fisher update`)
└── completions/            # mostly GITIGNORED; we track only vp.fish, vpr.fish
```

## WHERE TO LOOK

| Task | Location |
|------|----------|
| Add an alias | `conf.d/zz-aliases.fish` (loads last, so it wins over plugin-git) |
| Add env var | `conf.d/env.fish` (`set -gx NAME value`) |
| Add to PATH | `conf.d/paths.fish` (`fish_add_path`; `--move --append` for low priority) |
| Add a tool init (`foo init fish`) | new `conf.d/<tool>.fish`, gated `if type -q foo` |
| Add a fish plugin | `fisher install <owner/repo>` → updates `fish_plugins` |
| Add a completion we author | `completions/<cmd>.fish` + a `!` re-include in root `.gitignore` |
| Change the prompt | `../starship/starship.toml` (not here — shared with zsh) |

## CONVENTIONS

- **`config.fish` stays minimal** — greeting off only. Everything else in `conf.d/`.
- **Load order is alphabetical.** To override something a plugin sets (e.g. git
  aliases from `plugin-git`'s `conf.d/git.fish`), name your file so it sorts later
  (`zz-aliases.fish`).
- **Gate tool inits**: `if type -q <tool>; <tool> init fish | source; end` — so a
  missing tool doesn't error on startup.
- **PATH via `fish_add_path`**, never manual `set PATH`. Use `--move` because
  `fish_user_paths` is universal and won't reorder an existing entry otherwise.
- **`set -gx`** for exported globals.
- **Fish has autosuggestions + syntax highlighting natively** — don't add zsh-style
  plugins for those.

## ANTI-PATTERNS

- Heavy work or tool inits in `config.fish` (use `conf.d/`).
- Appending `foo completion fish` to `config.fish` — put it in `completions/` (lazy) or let brew's vendor dir provide it.
- Editing fisher-managed files (`functions/*`, `conf.d/git.fish`, most of `completions/`) — they're regenerated by `fisher update`.
- Prepending `.bun/bin`/`.cargo/bin` in `paths.fish` — they're appended so brew wins (e.g. brew `prettier`/`tsc` over bun's dependency copies).

## KEY ALIASES (`zz-aliases.fish`)

| Alias | Expands to |
|-------|-----------|
| `cat` | `bat` |
| `ls` / `ll` | `eza --icons --git` / `eza -l --icons --git` |
| `find` / `grep` | `fd` / `rg` |
| `cd` | `z` (zoxide) |
| `gs`/`ga`/`gc`/`gp` | git status / add . / commit -m / push (override plugin-git) |
| `tf` | `tofu` |

## COMPLETIONS

- **Brew-installed tools self-complete** — brew drops fish completions into
  `/opt/homebrew/share/fish/vendor_completions.d/` which fish auto-loads (bun,
  kubectl, orbctl, git, gh, …). Nothing to do.
- **We only track** `completions/vp.fish` + `vpr.fish` (Vite+, dynamic — they query
  `vp` at completion time). Everything else in `completions/` is gitignored.

## NOTES

- `config.fish` also documents that fish provides autosuggestions/highlighting natively.
- `onepassword.fish` only sets up completions — 1Password's `plugins.sh` (biometric CLI plugins) is POSIX-sh and not fish-sourceable.
- `env.fish` sets `EDITOR=zed`; `~/.cargo/env` has no fish variant, so `paths.fish` adds `~/.cargo/bin` directly.
